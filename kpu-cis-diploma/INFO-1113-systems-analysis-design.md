# INFO-1113 — Systems Analysis & Design

**Program:** KPU Computer Information Systems Diploma  
**Topic Area:** Requirements Engineering & Process Documentation

## What This Course Covered

- Systems Development Lifecycle (SDLC): planning, analysis, design, implementation, maintenance
- Requirements gathering techniques: interviews, observation, document analysis
- Process modelling: Data Flow Diagrams (DFDs), use case diagrams
- System documentation and Standard Operating Procedure (SOP) writing
- Stakeholder communication and managing expectations across technical and non-technical audiences

## Key Skills Developed

- Translating vague business needs into clear, documented technical requirements
- Writing SOPs that non-technical users can follow without hand-holding
- Identifying gaps between what a system does and what it needs to do
- Scoping a project before committing to a timeline or resource estimate

## How This Applies to My Career

This course is most directly reflected in my work as **IT Project Coordinator at F12.net**. Managing end-to-end infrastructure projects required exactly the skills taught here — gathering requirements from clients and internal teams, documenting the scope before work begins, and maintaining communication with stakeholders throughout.

At **ESW IT**, client scoping for consulting engagements follows the same analysis approach: understand the environment, document the current state, identify the gap, define the solution. The SOP writing skills from this course also show up in knowledge base documentation I contribute to for both support and project delivery teams.

# INFO-2413 — System Development Project

**Program:** KPU Computer Information Systems Diploma  
**Topic Area:** Full Project Lifecycle Management

## What This Course Covered

- Complete SDLC from requirements through post-implementation review
- Project planning: Work Breakdown Structure (WBS), Gantt charts, critical path analysis
- Requirements documentation: functional vs. non-functional requirements, user stories
- System design: architecture diagrams, interface mockups, data flow models
- Testing: unit testing, integration testing, user acceptance testing (UAT)
- Deployment planning and change management
- Post-implementation review: KPIs, lessons learned, stakeholder sign-off

## Key Skills Developed

- Owning a project end-to-end — from initial scoping through client sign-off
- Managing scope creep through clear documented requirements
- Communicating status and risks to both technical and non-technical stakeholders
- Conducting retrospectives and documenting lessons learned for future projects

## How This Applies to My Career

This is the capstone course of the program and the one most directly reflected in my **IT Project Coordinator role at F12.net**. Managing infrastructure projects — workstation rollouts, network refreshes, server migrations — follows exactly this lifecycle. The client scopes, I gather requirements, we plan resources and timelines, execute with the technical team, and close with a sign-off.

At **ESW IT**, the same structure applies to consulting engagements. Before recommending a solution, I scope the current environment, document findings, propose a structured approach, and present it to both technical and business stakeholders. INFO-2413 gave me the framework that makes this a professional process rather than an ad hoc one.

# INFO-1213 — Web Application Development

**Program:** KPU Computer Information Systems Diploma  
**Topic Area:** Web Technologies & Server Configuration

## What This Course Covered

- HTML structure and semantics: elements, attributes, forms, tables, multimedia
- CSS styling: selectors, box model, layout, responsive design basics
- Web server fundamentals: how HTTP requests and responses work, status codes
- IIS (Internet Information Services) setup and configuration on Windows Server
- Web application architecture: client-server model, static vs. dynamic content
- Introduction to server-side concepts and file permissions for web directories

## Key Skills Developed

- Reading and understanding HTML/CSS to diagnose web-related IT support issues
- Configuring and troubleshooting IIS on Windows Server
- Understanding the request-response cycle well enough to diagnose web access issues
- Managing web server permissions and virtual directories

## How This Applies to My Career

IIS configuration and troubleshooting came up directly in my work at **F12.net**, particularly in environments where line-of-business applications are hosted internally on Windows Server. Understanding how IIS serves content, handles authentication, and manages application pools is necessary for diagnosing application access issues.

The Remote Desktop Gateway (RD Gateway) configuration work I have done also builds on web server concepts — RD Gateway uses HTTPS and certificate-based authentication, so understanding web server architecture made those deployments more approachable. This course also reinforced why HTTPS matters in enterprise environments, which connects directly to the certificate management and security work at **ESW IT**.

# INFO-2312 — Database Systems

**Program:** KPU Computer Information Systems Diploma  
**Topic Area:** Relational Databases & SQL

## What This Course Covered

- Relational database concepts: tables, rows, columns, primary keys, foreign keys
- Entity-Relationship (ER) diagrams and database design
- Data normalization: 1NF, 2NF, 3NF — eliminating redundancy and anomalies
- SQL fundamentals: SELECT, INSERT, UPDATE, DELETE, JOINs (INNER, LEFT, RIGHT, FULL)
- Aggregate functions: COUNT, SUM, AVG, GROUP BY, HAVING
- Stored procedures, views, and basic indexing for performance

## Key Skills Developed

- Designing a well-structured relational database from requirements
- Writing SQL queries to retrieve and manipulate data across multiple tables
- Reading and modifying existing database schemas for reporting or maintenance
- Understanding how data relationships work in enterprise software systems

## How This Applies to My Career

Database understanding shows up in IT work in ways that are not always obvious. At **F12.net**, asset management systems and IT Glue documentation both use structured data models — understanding how relational data works makes it easier to build and maintain documentation that actually serves the team rather than becoming a disorganized mess.

SQL knowledge is also useful when IT is asked to extract data from line-of-business applications for reporting, audit, or migration purposes. I have used basic SQL knowledge to assist with data extraction tasks and to understand how ticketing and monitoring platforms store their data.
